## 検索手法

| 用語                             | 説明                                                                                                                                                                         |
| -------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Embedding 検索**               | テキストをベクトル化し、コサイン類似度で意味的な近さを測る検索手法。「エージェント」と「Agent」のようにクロスランゲージの類似も捉えられる                                    |
| **Lexical 検索**                 | クエリのトークンが文書内に何回出現するかで一致度を測る検索手法。字句的一致のみを行い、同義語・意味的類似は扱わない                                                           |
| **Hybrid 検索**                  | Embedding 検索と Lexical 検索の両スコアを統合して最終ランキングを決める手法。現在は RRF による融合を採用                                                                     |
| **BM25**                         | Lexical 検索のスコアリング手法。単純な出現頻度（TF）に加えて IDF（逆文書頻度）と文書長正規化を組み合わせ、一般的なトークンの過大評価を防ぐ。Robertson & Walker (1994) が提案 |
| **RRF** (Reciprocal Rank Fusion) | 2 つの順位リストを `1/(k + rank)` で点数化して合算する融合手法。スコアのスケール差に依存せず安定した融合ができる。デフォルト `k=60`                                          |
| **LLM Rerank**                   | Hybrid 検索の上位候補を LLM に渡し、クエリとの関連度順に再並べ替えする後処理ステップ                                                                                         |
| **emb fallback**                 | Lexical ヒット率が `EMB_FALLBACK_LEX_RATIO`（既定 5%）未満のとき、RRF を使わず Embedding スコア順のみで結果を返すフォールバック動作                                          |

---

## スコア・ランク

| 用語          | 説明                                                                                                                             |
| ------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| **emb_score** | Embedding 検索が算出したコサイン類似度スコア。0〜1 の範囲。値が高いほどクエリと意味的に近い                                      |
| **lex_score** | Lexical 検索が算出した BM25 スコア。0 以上の実数。値が高いほどクエリのトークンと字句的に一致している                             |
| **lex**       | lex_score の省略形。「lex=0」はその chunk に字句一致がまったくないことを意味する                                                 |
| **emb_rank**  | Embedding 検索結果での順位。1 が最上位                                                                                           |
| **lex_rank**  | Lexical 検索結果での順位。1 が最上位。lex=0 の chunk はランク外（RRF では `∞` 扱い）                                             |
| **RRF score** | RRF 融合後のスコア。`1/(k + emb_rank) + 1/(k + lex_rank)` で算出。lex_rank=∞ のときは Embedding 成分のみになる                   |
| **lex_ratio** | Embedding 結果件数に対する Lexical ヒット件数の比率。`len(lex_results) / len(emb_results)` で算出。emb fallback の判定に使用する |

## BM25 パラメータ

| 用語                                 | 説明                                                                                                                                    |
| ------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| **TF** (Term Frequency)              | クエリのトークン `t` が文書 `d` 内に出現する回数                                                                                        |
| **IDF** (Inverse Document Frequency) | トークン `t` の希少性を表す重み。多くの文書に出現するトークンほど低い値になる。Robertson IDF 式: `log((N - df + 0.5) / (df + 0.5) + 1)` |
| **df** (Document Frequency)          | トークン `t` が出現する文書（chunk）の件数                                                                                              |
| **N**                                | 検索対象の全 chunk 件数                                                                                                                 |
| **avgdl**                            | 全 chunk の平均文書長（文字数）                                                                                                         |
| **k1**                               | TF の飽和係数。同じトークンが何度出現してもスコアが無限に増えないよう制御する。既定値 1.5                                               |
| **b**                                | 文書長正規化係数。長い文書が有利にならないよう調整する。0=正規化なし、1=完全正規化。既定値 0.75                                         |

## インデックス・チャンク

| 用語                | 説明                                                                                                                    |
| ------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| **chunk**           | 見出し境界で分割した Markdown の断片。`chunk_id`・`path`・`heading_path`・`body`・`embedding` を持つ                    |
| **chunk_id**        | chunk のハッシュベース識別子。同一ファイル内で衝突した場合はサフィックス（`-2` など）を付与する                         |
| **heading_path**    | chunk が属する見出し階層のリスト。例: `["設計書", "6. 設計判断", "6.4 Lexical 検索"]`                                   |
| **body**            | chunk のテキスト本文。Embedding と BM25 スコアリングの対象                                                              |
| **min_chunk_level** | チャンク分割の最小見出しレベル。既定値 6（h1〜h6 すべてが分割境界）。値を小さくすると粒度が粗くなる                     |
| **doc_type**        | specs カテゴリ内の文書種別。`requirement` / `design` / `plan` など。`.doc_structure.yaml` の `doc_types_map` で定義する |

## ID・パターン

| 用語            | 説明                                                                                                                          |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **ID_RE**       | ID 形式（例: `FNC-001`、`DES-026`）を検出する正規表現 `[A-Z]+-\d+`。完全一致時に BM25 スコアへ固定ボーナス (+10.0) を加算する |
| **ID ブースト** | ID 形式のクエリトークンが chunk 内に存在するときに加算される固定スコア。通常の BM25 スコアに上乗せされる                      |

## 品質評価

| 用語                       | 説明                                                                                                                           |
| -------------------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| **ゴールデンセット**       | 検索品質を測定するための正解付きクエリ集合。`meta/test_docs/` 配下で管理（git 管理外）                                         |
| **recall**                 | クエリに対して正解文書が結果に含まれた割合。doc-db の主目標指標                                                                |
| **正答率**                 | ゴールデンセットの全クエリのうち、上位 N 件内に正解が含まれたクエリの割合                                                      |
| **EMB_FALLBACK_LEX_RATIO** | emb fallback を発動するための lex_ratio 閾値。既定値 0.05（5%）。lex_ratio がこの値未満のとき Embedding スコア順のみを使用する |
