# NFR-004: doc-db 検索品質 要件定義書

## 概要

doc-db の検索品質に関する要求を定義する。doc-advisor の ToC 方式・現行 Embedding 方式を品質ベースラインとし、それと同等以上の recall を達成することを目標とする。

## 前提条件

- 評価機能（FNC-007）が動作する
- ゴールデンセットが利用可能である

## 要件一覧

| ID     | 要件                                                                                                 |
| ------ | ---------------------------------------------------------------------------------------------------- |
| QUL-01 | doc-db の recall は doc-advisor の ToC 方式・現行 Embedding 方式と同等以上である                     |
| QUL-02 | false negative の傾向が分類され、原因（ID 検索失敗 / 長文希釈 / 自然文の意味ズレ等）が特定されている |
| QUL-03 | 結果は説明可能である（どの chunk・どのスコアでヒットしたかをユーザーが追跡できる）                   |
| QUL-04 | ToC 方式 / doc-db 方式の採否を判断できる集計値が、ゴールデンセット評価で得られる                     |

> recall 同等基準の正式な測定契約（到達ライン・算出式・top-N・対象ゴールデンセット版・比較対象方式の固定方法・検索設定・集計方法）と、検索時間・Index 生成時間・API コスト等の性能/コスト数値目標は **評価計画** で確定する。要件定義では扱わない。

## 品質目標の運用方針

- recall・precision は数値目標として運用するが、`plugins/forge/docs/spec_priorities_spec.md §3` に従い、構造品質（責務分割・境界）の数値化はしない
- 検索方式の良し悪しはゴールデンセット上の数値で判定する
