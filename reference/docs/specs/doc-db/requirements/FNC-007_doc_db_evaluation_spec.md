# FNC-007: doc-db 評価機能 要件定義書

## 概要

ゴールデンセットを用いて doc-db の検索品質を測定し、ToC 方式・現行 Embedding 方式（doc-advisor）と比較できるレポートを生成する。本機能の主目的は **doc-db 方式が doc-advisor 方式を置き換える価値を持つかを判断する** ことである。

## 前提条件

- ゴールデンセット（`meta/test_docs/`）が利用可能である
- ToC 方式・現行 Embedding 方式（doc-advisor）が同一ゴールデンセットで動作する状態にある
- doc-db の Index が生成済みである

## 要件一覧

### 操作

| ID    | 要件                                                                                |
| ----- | ----------------------------------------------------------------------------------- |
| OP-01 | ユーザーは評価 skill を実行して、ゴールデンセット全クエリの評価レポートを生成できる |
| OP-02 | ユーザーは比較対象の方式（ToC / 現行 Embedding / doc-db）を選択できる               |
| OP-03 | ユーザーは個別クエリの結果を確認でき、期待結果との差分を確認できる                  |

### 評価指標

| ID     | 要件                                                                                                      |
| ------ | --------------------------------------------------------------------------------------------------------- |
| MET-01 | 検索品質指標として recall・precision・false negative 数を測定する                                         |
| MET-02 | 性能指標として検索時間と Index 生成時間を測定する                                                         |
| MET-03 | コスト指標として Embedding API・LLM Rerank API の呼び出し回数および消費 token 数を API 種別ごとに測定する |

### レポート

| ID     | 要件                                                                                                    |
| ------ | ------------------------------------------------------------------------------------------------------- |
| REP-01 | レポートは方式ごとの集計値（recall / precision / 検索時間 / コスト）を表形式で提示する                  |
| REP-02 | レポートは false negative の発生クエリと対象文書を一覧化し、傾向（ID 検索失敗・自然文失敗等）に分類する |
| REP-03 | レポートは Markdown 形式で出力する                                                                      |

> 評価指標の正式定義（recall の算出式・top-N・集計方法等）、incomplete Index の扱い、false negative 傾向の分類カテゴリ、API レート制限時の中断・再開仕様、評価実施環境の固定方針は **評価計画** で確定する。要件定義では扱わない。
