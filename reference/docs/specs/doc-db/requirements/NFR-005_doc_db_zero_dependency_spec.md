# NFR-005: doc-db 配布制約 要件定義書

## 概要

doc-db は配布の容易さを担保するため、ユーザー環境への追加要求を最小化する。具体的には次の 3 軸で配布制約を定義する: (1) Python 標準ライブラリのみで動作する（pip install 不要）、(2) 常駐サーバープロセスを必須としない、(3) Embedding API および LLM Rerank API の提供者として OpenAI を採用し、他ベンダーの API key を追加で必須としない。doc-advisor の NFR-003（標準ライブラリ優先方針）を継承する。

## 前提条件

- 実行環境に Python 3 がインストールされている
- ネットワーク経由で OpenAI API にアクセスできる

## 要件一覧

| ID     | 要件                                                                                                                                 |
| ------ | ------------------------------------------------------------------------------------------------------------------------------------ |
| DEP-01 | doc-db は Python 標準ライブラリのみで動作する                                                                                        |
| DEP-02 | ユーザーに pip install を要求しない                                                                                                  |
| DEP-03 | 常駐サーバープロセス（外部 DB サーバー・検索エンジンサーバー等）を必須としない                                                       |
| DEP-04 | Embedding API および LLM Rerank API の提供者として OpenAI を採用し、他ベンダーの API key を追加で必須としない（OpenAI 1 種類で完結） |

> 永続化形式・Lexical 検索の実装方式・reranker の選定は設計書で決定する。本要件は「ユーザー環境への要求」のみを規定する。

## 制約の理由

- 配布形態を簡素化し、`/plugin install` だけで利用可能にする
- doc-advisor の NFR-003 と整合させ、bw-cc-plugins 全体のポリシーを維持する
- 環境依存（外部 wheel のビルド・追加サーバー起動・別ベンダー API key 取得）を排除する
