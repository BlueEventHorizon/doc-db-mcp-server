# COMMON-REQ-001: 段階的バージョンマイグレーション要件定義書

**作成日**: 2026-03-15
**作成者**: k_terada

## 概要

バージョン付きデータ形式の後方互換性を保証するための、段階的マイグレーションフレームワークを定義する。
旧バージョンのデータを現行バージョンに変換し、アプリケーションが正常に動作できるようにする。

## 適用範囲

本要件は、以下の条件を満たすあらゆるデータ形式に適用できる:

- メジャーバージョン番号を持つ
- バージョン間で構造が変化する
- 旧バージョンのデータが存続する環境で新バージョンのアプリケーションが動作する必要がある

### 適用例

| データ形式            | バージョン識別方法                      | 適用箇所                                         |
| --------------------- | --------------------------------------- | ------------------------------------------------ |
| `.doc_structure.yaml` | `# doc_structure_version: X.0` コメント | `plugins/forge/scripts/migrate_doc_structure.py` |

## 機能要件

### FR-01: バージョン検出

| ID      | 要件                                                       |
| ------- | ---------------------------------------------------------- |
| FR-01-1 | データからバージョン番号を検出する仕組みを備える           |
| FR-01-2 | バージョン識別子がない場合は最古のバージョン（v1）と見なす |
| FR-01-3 | バージョン番号はメジャーバージョン（整数）を使用する       |

### FR-02: 段階的マイグレーション

| ID      | 要件                                                                                           |
| ------- | ---------------------------------------------------------------------------------------------- |
| FR-02-1 | マイグレーションは1バージョンずつ段階的に適用する（例: v1→v2→v3。v1→v3 の直接変換は行わない）  |
| FR-02-2 | マイグレーション関数はレジストリ（辞書等）に登録する。キーはターゲットバージョン、値は変換関数 |
| FR-02-3 | 適用対象は `detected_version < v <= current_version` の範囲を昇順に実行する                    |
| FR-02-4 | 各マイグレーション関数は前段の出力を入力として受け取る（チェーン実行）                         |
| FR-02-5 | 現行バージョンは定数として定義する                                                             |

### FR-03: マイグレーション関数の規約

| ID      | 要件                                                                                                              |
| ------- | ----------------------------------------------------------------------------------------------------------------- |
| FR-03-1 | 各マイグレーション関数は「1つ前のバージョンの出力形式」を入力として受け取り、「ターゲットバージョンの形式」を返す |
| FR-03-2 | マイグレーション関数は冪等であること（同じ入力に対して同じ出力を返す）                                            |
| FR-03-3 | マイグレーション関数は副作用を持たないこと（入力データの変換のみ。ファイル I/O、ネットワーク通信等を行わない）    |

### FR-04: エラーハンドリング

| ID      | 要件                                                                                                                                                                                           |
| ------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| FR-04-1 | マイグレーション関数は副作用を持たない純粋関数（FR-03-3）であり、例外はコードバグとして伝播させる。サイレントフォールバック（`except Exception: return original`）で不具合を隠蔽してはならない |
| FR-04-2 | 検出バージョンが current_version より大きい（未知の将来バージョン）場合、マイグレーションをスキップしそのまま使用する                                                                          |
| FR-04-3 | バージョン検出に失敗した場合は最古のバージョンとして扱う（FR-01-2 と同様）                                                                                                                     |

### FR-05: 拡張性

| ID      | 要件                                                                                                    |
| ------- | ------------------------------------------------------------------------------------------------------- |
| FR-05-1 | 新バージョン追加時はマイグレーション関数を1つ登録し、current_version 定数を更新するだけで対応できること |
| FR-05-2 | 既存のマイグレーション関数を変更する必要がないこと（Open-Closed 原則）                                  |

## 非機能要件

| ID     | 要件                                                                           |
| ------ | ------------------------------------------------------------------------------ |
| NFR-01 | マイグレーションはメモリ上のデータ変換のみで完結する（元データを書き換えない） |
| NFR-02 | 外部ライブラリに依存しない                                                     |

## 設計パターン

```python
CURRENT_VERSION = 3

MIGRATIONS = {
    2: migrate_v1_to_v2,
    3: migrate_v2_to_v3,
}

def apply_migrations(data, detected_version):
    """段階的マイグレーションを適用する"""
    if detected_version >= CURRENT_VERSION:
        return data  # FR-04-2: 未知のバージョンはスキップ

    targets = [v for v in sorted(MIGRATIONS.keys())
               if detected_version < v <= CURRENT_VERSION]

    # FR-04-1: マイグレーション関数は純粋関数（FR-03-3）であり、
    # 例外はコードバグとして伝播させる。サイレントフォールバックで隠蔽しない。
    for v in targets:
        data = MIGRATIONS[v](data)

    return data
```

### 新バージョン追加手順

1. `migrate_vN_to_vN1()` 関数を実装する
2. `MIGRATIONS` に `{N+1: migrate_vN_to_vN1}` を登録する
3. `CURRENT_VERSION` を `N+1` に更新する
4. 既存のマイグレーション関数は変更しない

## 用語定義

| 用語                       | 定義                                                                                                                               |
| -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| 段階的マイグレーション     | v1→v2→v3 のように1バージョンずつ順番に変換を適用する方式。各ステップが独立しており、中間バージョンの変換ロジックが確実にテスト可能 |
| マイグレーションレジストリ | ターゲットバージョンとマイグレーション関数の対応を保持する辞書。新バージョン追加時のエントリポイント                               |
| current_version            | アプリケーションが対応する最新のデータ形式バージョン                                                                               |
| detected_version           | 入力データから検出されたバージョン番号                                                                                             |
