# FNC-003 インデックス生成の高速化 要件定義書

## 概要

文書検索用のインデックス生成を、現行の ToC 生成パイプライン（AI による並列解析）より高速に実行する。

## 前提条件

- `.doc_structure.yaml` が設定済みであること
- 対象文書がスキャン対象ディレクトリに存在すること

## 要件一覧

### 速度要件

- インデックスの全体再構築が、現行 ToC 生成パイプラインより高速に完了する
- 差分更新（変更・追加・削除された文書のみ）に対応する
- 差分更新は全体再構築より高速に完了する

### インデックス生成のトリガー

- create-specs-toc / create-rules-toc スキル実行時に生成される
- 文書の追加・変更・削除を検出し、差分のみ更新する

### インデックスの管理

- インデックスは `.claude/doc-advisor/` 配下に永続化される
- 差分検出には既存のチェックサム機構（SHA-256 ハッシュ）を活用する

### エラーケース

| 条件                             | 動作                                   |
| -------------------------------- | -------------------------------------- |
| インデックスが破損している       | 全体再構築を実行する                   |
| 差分検出中にファイルが削除された | インデックスから該当エントリを削除する |

## 関連要件

- FNC-001: コンテキスト外検索（インデックスを使って検索を実行）
- NFR-002: YAML ToC の廃止（インデックスが ToC の役割を代替）
