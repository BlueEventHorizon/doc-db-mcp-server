# NFR-002 YAML ToC の廃止 要件定義書

> **⚠️ 凍結（2026-04-11）**: 本要件は凍結状態。v0.2.0 の品質テスト（ゴールデンセット 43 クエリ比較）で ToC の検索精度が Embedding Index を上回ることが確認された。廃止の前提条件（Embedding が ToC と同等以上の精度）を満たさないため、ToC 廃止は実施しない。現行アーキテクチャは ToC を主軸、Index を補完とするハイブリッド方式（DES-006 v2.0 参照）。将来 Embedding 精度が ToC を上回ることが確認された場合に再検討する。

## 概要

新方式の検索インデックスが十分な精度を達成した後、現行の YAML ToC とその生成パイプラインを廃止する。

## 前提条件

- FNC-002（見落としゼロの検索精度）の精度検証が完了していること

## 要件一覧

### 廃止条件

- FNC-002 の精度検証で見落とし 0 件が確認されていること
- 精度検証完了後の運用期間中に、インデックス生成失敗・検索エラーが発生していないこと

### 廃止対象

- `{category}_toc.yaml`（specs_toc.yaml / rules_toc.yaml）
- ToC 生成パイプライン:
  - pending YAML テンプレート生成
  - toc-updater エージェントによる AI 解析（並列実行）
  - pending YAML のマージ
  - ToC 検証
- `.toc_work/` ディレクトリ機構

### 維持対象

- `.toc_checksums.yaml`（差分検出に引き続き使用）
- `.doc_structure.yaml`（文書構造の定義）
- query-specs / query-rules スキルの公開契約（入出力形式は現行互換を維持）
- create-specs-toc / create-rules-toc スキル（インデックス生成方式のみ変更）
- staleness check 機能（インデックスの freshness check に置換）

### 運用継続性要件

- 新方式は、現行 ToC パイプラインが担う以下の運用機能と同等の継続性を確保する:
  - 差分更新（変更・追加・削除の反映）
  - インデックス破損時の復旧（全体再構築）
  - 処理中断時の再開

### エラーケース

| 条件                                 | 動作             |
| ------------------------------------ | ---------------- |
| 精度検証が未完了の状態で廃止を試みた | 廃止を実行しない |

## 関連要件

- FNC-002: 見落としゼロの検索精度（廃止判断の前提）
- FNC-003: インデックス生成の高速化（ToC 生成パイプラインの代替）
- REQ-001 FR-02-3: ToC YAML 出力（廃止対象）
- REQ-001 FR-05-1/2: ToC 検索（検索方式のみ変更）
