# FNC-002 見落としゼロの検索精度 要件定義書

## 概要

doc-advisor の文書検索において、関連する文書を 1 件も見落とさない検索精度を実現する。返却する文書の件数に上限を設けず、必要な文書は全て返す。

## 前提条件

- FNC-001（コンテキスト外検索）が実装されていること
- 検索インデックスが最新の状態であること

## 要件一覧

### 精度要件

- 関連する文書は全て検索結果に含まれる（False Negatives = 0）
- 件数制限を設けない。関連文書が 50 件あれば 50 件全て返す
- 現行の AI による ToC 全量検索と同等以上の精度を達成する

### 検索対象のフィールド

- タイトル（title）
- 目的（purpose）
- 内容詳細（content_details）
- 適用タスク（applicable_tasks）
- キーワード（keywords）

### カテゴリ分離要件

- 現行の rules / specs のカテゴリ分離を維持する
- 各カテゴリのインデックスは独立して生成・検索される

### 精度検証要件

- 精度を測定するための固定テストクエリセット（ゴールデンセット）を用意する
- ゴールデンセットの正解データは人手で確認・作成する（現行 AI 検索の結果は初期ベースラインとして使用し、人手レビューで補正する）
- 新方式の結果とゴールデンセットを比較し、見落とし件数を測定する
- 見落とし 0 件を確認してから本番適用する

### 品質テスト結果（v0.2.0 実施）

v0.2.0 でゴールデンセット（43 クエリ: rules 21 件 + specs 22 件、6 種別: direct/task/crosscut/synonym/proper/negative）を使用した品質テストを実施:

- **ToC（`--toc`）**: 最も精度が高い。AI による YAML メタデータ全量マッチングにより false negative が最小
- **Index（`--index`）**: 良好な精度。ToC メタデータが古い文書で ToC が見落とすケースを補完
- **auto (v0.2.x: Index ∪ ToC union)**: 最良の網羅性。両方の候補を union し AI が最終判定

テスト基盤: `meta/test_docs/` に配置（git 管理外）

### 品質基準（v0.3.x: auto = doc-advisor 単独完結（ToC + Embedding union））

v0.3.x で auto モードは **doc-advisor 単独完結（ToC 常時実行 + API キーがあれば Embedding を並列実行し集合 union でマージ）** に再定義された（DES-006 §2.3 / §10.6 Phase 5、forge:DES-001_forge_query_abstraction_design §1.2）。本 FNC-002 の見落としゼロ要件は引き続き適用する。

- ゴールデンセットによる recall 測定で auto モード（ToC ∪ Embedding union）が `--toc` モード単独と **同等以上の recall** を達成すること
- 達成しない場合は doc-advisor 側の Embedding スコア閾値（`search_docs.py --threshold`）や ToC メタデータ（`content_details` / `keywords` / `applicable_tasks`）の網羅性を調整して再評価する
- `--toc` モードは外部依存ゼロ・最高精度の参照実装として維持し、auto モードの **下限基準** として使用する（auto は ToC を常時実行するため、原理上 `--toc` モードの recall を下回らない）
- doc-db plugin の Hybrid 検索を使う場合は forge 側 `/forge:query-db-*` でバックエンドとして doc-db が採用された際の挙動であり、doc-advisor の auto モードとは別経路（本 FNC-002 の対象外）

### エラーケース

| 条件                                     | 動作                                                                                                                               |
| ---------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| インデックスが古い（文書変更後に未更新） | 警告を表示し、インデックス再生成を案内する。再生成完了までは検索を実行しない（古いインデックスでの検索は見落としリスクがあるため） |
