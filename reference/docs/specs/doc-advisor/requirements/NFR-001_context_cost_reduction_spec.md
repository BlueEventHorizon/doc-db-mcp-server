# NFR-001 コンテキストコスト削減 要件定義書

## 概要

文書検索時に AI のコンテキストウィンドウに読み込まれるデータ量を削減する。現行の ToC YAML 全量読み込みを不要にする。

## 前提条件

- FNC-001（コンテキスト外検索）が実装されていること

## 要件一覧

### コスト削減要件

コンテキストコスト削減の達成度はモードにより異なる:

| モード    | コンテキスト消費                                                                                  | 削減度                                  |
| --------- | ------------------------------------------------------------------------------------------------- | --------------------------------------- |
| `--index` | 候補パスリスト + 候補文書本文のみ。ToC 読み込みなし                                               | **50% 以上削減（目標達成）**            |
| `--toc`   | ToC YAML 全量 + 候補文書本文（現行と同等）                                                        | 削減なし                                |
| `auto`    | ToC エントリ ≤100: ToC 全量 + Index 候補 + 候補本文 / >100: Index 候補で絞った縮小 ToC + 候補本文 | ≤100: 現行と同等〜微増 / >100: 大幅削減 |

> **設計判断**: auto モードは精度優先（false negative の最小化）が原則。ToC エントリが 100 件以下のときは ToC を全量 Read して最高精度を維持する。100 件超のときは AI が ToC を全量読みきれないため、Index 検索（`search_docs.py --threshold 0.2`）で広めに候補を絞り、`filter_toc.py` でその候補に対応する縮小 ToC だけを Read する。コスト削減が必要な場合は `--index` モードを明示指定する。

### 測定方法

- 測定単位: トークン数（AI のコンテキストウィンドウ消費量）
- 測定条件: 固定クエリセット（FNC-002 のゴールデンセットと共通）での平均値
- `--index` モードで: 候補パスリストのトークン数 + 候補文書の本文読み込みトークン数
- `--toc` モードで: ToC YAML + 候補文書の本文読み込みトークン数（現行と同等）
- `--index` モードで 50% 以上削減されることを確認する

### エラーケース

| 条件                                           | 動作                                                                                                                                                   |
| ---------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 検索インデックスが利用不可（`--index` モード） | エラーを通知する。ToC にフォールバックしない                                                                                                           |
| ToC が存在しない（`--toc` モード）             | エラーを通知する。Index にフォールバックしない                                                                                                         |
| 両方利用不可（`auto` モード）                  | エラーを通知し、create-toc 実行 or API キー（`OPENAI_API_DOCDB_KEY` 推奨、未設定時は `OPENAI_API_KEY` にフォールバック。DES-007 統一仕様）の設定を案内 |
