# FNC-001 コンテキスト外検索 要件定義書

## 概要

doc-advisor の文書検索（query-specs / query-rules）において、検索処理を AI のコンテキストウィンドウの外で実行し、AI には検索結果（候補パスのリスト）のみを渡す。

`query-specs` / `query-rules` SKILL は fork 型 SKILL (`context: fork`) として実行され、fork 型 SKILL は fork 内で検索を完結させ、最終結果のみを親 Claude に return する（fork 型 SKILL 内完結フロー、DES-006 §2.4 参照）。

## 前提条件

- doc-advisor プラグインがインストールされていること
- `.doc_structure.yaml` が設定済みであること
- 対象文書がスキャン対象ディレクトリに存在すること
- OpenAI API key が環境変数で利用可能であること（`auto` モード / `--index` モード時）。参照規約は FNC-004 / DES-007 を参照

## 要件一覧

### 機能要件

- `--index` モードでは、文書の検索処理が AI のコンテキストウィンドウを消費せずに実行される（`search_docs.py` / `grep_docs.py` による doc-advisor Embedding 検索）
- `auto` モードでは、**doc-advisor 単独完結** の検索が AI のコンテキスト外で実行される。ToC キーワード検索を常時実行し、API キー（`OPENAI_API_DOCDB_KEY` または `OPENAI_API_KEY`）が設定されている場合のみ doc-advisor Embedding 検索も並列実行して結果を **集合 union** でマージする（DES-006 §2.3 / §2.4 / §6.1 参照）。**doc-db plugin は呼ばない**（検索バックエンド選択は forge 側 `/forge:query-db-*` の責務。forge:DES-001_forge_query_abstraction_design §1.2 / §2.3 参照）
- `--toc` モードおよび `auto` モードの ToC 検索パスでは、AI が ToC YAML を Read してキーワードマッチングを行う（コンテキストを消費するが、高精度）
- 検索結果は候補文書のパスリストとして AI に渡される
- AI は受け取ったパスリストの文書を Read して内容を確認する
- query-specs と query-rules の両方で同じ 3 モード検索方式が使用される
- ユーザーから見た実行方法と出力形式は変わらない（モード切替は `--toc` / `--index` オプション、省略時は `auto`）

### 検索の入力

- タスクの説明文（自然言語テキスト）

### 検索の出力

- AI に渡す最終出力: 候補文書のパスリスト（プロジェクトルート相対パス）
- 検索エンジン内部レスポンス: パスリストに加え、スコアまたは関連度情報を含む（内部利用、AI には渡さない）

### テスト要件

- 新規に作成する Python スクリプトには `tests/` 配下に自動テストを追加する

### エラーケース

| 条件                                                 | 動作                                                                                                                                                                          |
| ---------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 検索インデックスが存在しない                         | `--index` モード: エラー通知し再構築を案内する。`auto` モード: Embedding 検索を **静かに空リスト** として扱い ToC 結果のみで応答する                                          |
| 検索クエリが空                                       | エラーメッセージを返す                                                                                                                                                        |
| 該当文書が 0 件                                      | 「該当する文書が見つかりませんでした」を 0 件応答として return（エラーではない）                                                                                              |
| `auto` モード時に API キー未設定                     | Embedding 検索をスキップし、ToC 検索のみで応答する（`auto` モード全体は失敗にしない）                                                                                         |
| `auto` モード時の Embedding 検索失敗（API エラー等） | **静かに空リスト** として扱い、ToC 結果のみで応答する（`auto` モード全体は失敗にしない。Embedding を必須としたい場合は `--index` モードを明示する。DES-006 §2.3 / §6.1 参照） |
