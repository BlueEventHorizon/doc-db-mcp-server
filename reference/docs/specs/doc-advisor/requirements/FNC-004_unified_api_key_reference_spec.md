# FNC-004 統一 API KEY 参照規約 要件定義書

## 概要

OpenAI Embedding API を利用する全プラグイン（doc-advisor / doc-db 他）の Embedding 利用箇所において、API KEY の参照規約を統一する。

`OPENAI_API_DOCDB_KEY` を優先参照し、未設定時のみ `OPENAI_API_KEY` をフォールバックとして使用する。

## 前提条件

- OpenAI API key が環境変数のいずれかで設定されていること

## 要件一覧

### KEY 参照

| ID     | 要件                                                                                                                                                                 |
| ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| KEY-01 | OpenAI Embedding API を呼び出す全プラグインのスクリプトは、`OPENAI_API_DOCDB_KEY` を優先参照し、未設定時のみ `OPENAI_API_KEY` をフォールバックとして使用する         |
| KEY-02 | API KEY 未設定 / 認証エラー時のエラーメッセージ・案内文言は `OPENAI_API_DOCDB_KEY`（フォールバック: `OPENAI_API_KEY`）の双方を明示する                               |
| KEY-03 | 各プラグインは API KEY 解決ロジックを **独立実装** する（cross-plugin import を行わない）。プラグイン間の独立性および install 環境でのパス解決破壊リスクを避けるため |

### 互換性

| ID     | 要件                                                                                                                                    |
| ------ | --------------------------------------------------------------------------------------------------------------------------------------- |
| CMP-01 | 既存ユーザーが `OPENAI_API_KEY` のみを設定している場合、フォールバック経路で従来通り動作する（破壊的変更ではない）                      |
| CMP-02 | 既存ユーザーが `OPENAI_API_DOCDB_KEY` を新規に設定した場合、doc-advisor の Embedding 検索と doc-db Hybrid 検索が同一の KEY を共有できる |

### 適用範囲

| ID     | 対象                                                                                             |
| ------ | ------------------------------------------------------------------------------------------------ |
| SCP-01 | doc-advisor の Embedding 関連スクリプト: `embedding_api.py` / `search_docs.py` / `embed_docs.py` |
| SCP-02 | doc-db の Embedding 関連スクリプト: `embedding_api.py` 他、OpenAI API を呼び出す全モジュール     |
| SCP-03 | 各プラグインの SKILL.md 内のエラーメッセージ・案内文言（API KEY 関連）                           |
| SCP-04 | 将来追加されるプラグインで OpenAI Embedding API を利用するもの全て                               |

### テスト要件

| ID     | 要件                                                                                                                                                                                                                                                 |
| ------ | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| TST-01 | 各プラグインの `embedding_api.py`（または相当箇所）に対し、KEY 参照ロジックをユニットテスト化する。**`OPENAI_API_DOCDB_KEY` のみ設定 / `OPENAI_API_KEY` のみ設定 / 両方設定 / 両方未設定** の 4 ケースで正しいキー（または空文字列）が返ることを検証 |
| TST-02 | 環境変数モックの差し替えはテスト共通パターンとして既存テストを更新する。スクリプト本体の振る舞いは変えない                                                                                                                                           |

## 非機能要件

| ID     | 要件                                                                                                               |
| ------ | ------------------------------------------------------------------------------------------------------------------ |
| NFR-01 | 本規約適用後も、各プラグインの検索品質要件（doc-advisor: FNC-002、doc-db: NFR-004）を引き続き満たすこと            |
| NFR-02 | プラグイン間の独立性を保つため、API KEY 解決ロジックの共通実装を別プラグインや外部パッケージとして切り出さないこと |
